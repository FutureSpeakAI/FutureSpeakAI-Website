Your AI Chief of Staff — a desktop-native voice assistant built on Gemini Live, Claude Opus, and ElevenLabs. Agent Friday lives on your desktop, sees your screen, remembers your preferences across sessions, manages background tasks through a team of specialist sub-agents, controls your installed software, monitors the world in real-time, and can even join your video calls.

Voice Pipeline: Gemini Live WebSocket with bidirectional PCM audio at 24kHz. Gapless playback via Web Audio API scheduling.

Dual-LLM: Gemini handles real-time conversation; Claude Opus is invoked via tool call (ask_claude) for complex reasoning, code generation, and deep analysis.

# Create environment file

# Run in development mode

npm run build

# Package as installer + portable ZIP
npm run package
# Output: release/Agent Friday Setup 1.0.0.exe (installer)
#         release/Agent Friday-1.0.0-win.zip   (portable)

Key Features
Voice & Audio
Bidirectional WebSocket streaming at 24kHz PCM via Gemini Live. Audio captured from the microphone via AudioWorklet (with ScriptProcessorNode fallback). Playback uses precise Web Audio scheduling (source.start(exactTime)) for gapless chunk playback.

Memory System
Three-tier memory with automatic consolidation:

Tier	Retention	Example
Short-term	Current session	Recent conversation turns
Medium-term	Cross-session patterns	"User often asks about React" (promoted after 3+ occurrences)
Long-term	Permanent facts	"User's name is Stephen", "User prefers dark mode"
Memory extraction runs every 5 minutes. Episodic memory captures full session summaries with topics, emotional tone, and key decisions. Semantic search enables natural-language queries across all memory types.

Multi-Agent System
Friday orchestrates a team of specialist agents:

Agent	Voice	Role	Best For
Atlas	Adam (ElevenLabs)	Research Director	Deep research, analysis, fact-checking
Nova	Bella (ElevenLabs)	Creative Strategist	Writing, brainstorming, communications
Cipher	Daniel (ElevenLabs)	Technical Lead	Code review, architecture, debugging
Sub-agents run on Gemini 2.5 Flash (text mode) with ElevenLabs TTS for distinct voices. They execute in the background and speak their findings in their own voice.

Live Call Participation
Join video calls (Google Meet, Zoom, Teams) using virtual audio routing:

Friday's voice --> VB-Cable Virtual Mic --> Meeting app (as "microphone")
Meeting audio --> System speakers --> Friday's mic input
Requires VB-Cable (free). Select "CABLE Output" as the microphone in your meeting app.

Reconnection Hardening
Self-managed retry loop with:

Linear backoff (3s to 30s cap) over 15 attempts
Network-aware: pauses when offline, resumes on navigator.onLine
Conversation context preserved across reconnects via session summary
Voice/accent anchoring maintained through reconnection

Privacy & Security
API keys are stored locally in Electron's userData directory (encrypted at rest by the OS)
No telemetry — Agent Friday does not phone home or collect usage data
Screen capture is local-only — images are sent directly to Gemini for context and never stored
Memory is stored locally on disk in JSON format in your user data directory
Webcam access is tool-gated — the agent must explicitly request permission
Self-improvement changes require explicit user approval before any code is modified
Connector tools run locally on your machine — no data leaves your system except for API calls you initiate
Troubleshooting
Issue	Solution
No voice response	Check Gemini API key in Settings; ensure microphone permissions are granted
"MCP client failed to connect"	Install @anthropic-ai/desktop-commander globally or ensure npx works
Agents have no voice	Add an ElevenLabs API key in Settings
Can't join calls	Install VB-Cable and select "CABLE Output" as meeting mic
Connector tools missing	The connector auto-detects software at startup; restart the app after installing new software
World Monitor not available	Install World Monitor (npm install in its directory) — the connector detects it automatically
Build fails	Run npm run typecheck to identify TypeScript errors
Packaged app shows blank screen	Ensure npm run build succeeds before npm run package
